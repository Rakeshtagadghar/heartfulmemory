# Featurebase Free Tier And Widget Scope

As of March 6, 2026, sprint 43 should stay inside Featurebase's free-plan surface area. The pricing page is the primary source for scope: it says all plans include both suites, and the free plan explicitly includes live chat plus the product suite.

## Free-tier-safe decisions

- Use Featurebase Feedback widget for in-app bug and idea submission.
- Use Featurebase Changelog widget for "What's new" entry points.
- Use Featurebase Messenger for quick in-app support and feedback entry.
- Keep default Featurebase branding in place. Branding removal is a paid add-on, so this sprint should not assume white-labeling.
- Keep the embedded portal optional at first because it is a lower-priority sprint item, not because of a pricing blocker.

## Planned placements

- Studio layout: add launcher entry points that do not overlap the canvas or intercept drag gestures.
- Dashboard and create flow screens that render inside `AppShell`: add visible feedback and changelog entry points.
- Account/settings surfaces: add feedback and changelog links after the shared loader and launchers are stable.
- Marketing pages: do not load Featurebase by default.

## Free plan snapshot

- 1 seat
- Live chat
- Unlimited conversations
- Unified inbox and ticketing
- Help Center (50 articles)
- Feedback and roadmaps
- Surveys
- Product Suite included
